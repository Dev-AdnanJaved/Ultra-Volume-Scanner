REPLIT REQUEST — ADD PAPER TRADING MODE FOR STRATEGY TESTING
=============================================================

Please add a paper trading mode that simulates the complete
exit strategy using real Binance market data but zero real orders.
This lets us verify every rule works correctly before going live.

══════════════════════════════════════════════════════════════
WHAT TO ADD
══════════════════════════════════════════════════════════════

Add this to config.json under "trading":
  "paper_mode": true

When paper_mode = true:
  - ALL Binance order calls (place_order, cancel_order, etc.)
    are SKIPPED — no real orders placed
  - Real Binance PRICE DATA is still fetched (read-only)
  - A fake paper portfolio is tracked in data/paper_trades.json
  - Every action that would happen in live trading is
    LOGGED to Telegram with prefix 📝 PAPER:

══════════════════════════════════════════════════════════════
HOW TO ENABLE
══════════════════════════════════════════════════════════════

config.json:
{
  "trading": {
    "enabled": true,        ← must be true
    "paper_mode": true,     ← this makes it paper only
    "margin_value": 2,
    "leverage": 5,
    "sl_pct": 15.0,
    "max_open_trades": 5
  },
  "exit_strategy": {
    "enabled": true         ← must be true
  }
}

When both enabled=true AND paper_mode=true:
  → All strategy logic runs exactly as live
  → All scoring, SL moves, partial closes, time limits run
  → Zero real orders placed on Binance
  → Everything logged to Telegram with 📝 PAPER prefix

WHAT THIS LETS US VERIFY
══════════════════════════════════════════════════════════════

1. SL placement — see exact SL price in Telegram on every entry

5. TP20 SL — see it always move to +12%
6. TP30 trail — see trail activate and update as price rises
7. Time limits — see market close when 48h/72h expires
8. Emergency exits — trigger by manually setting BTC to dump
9. Position sizing — see fake $ amounts per trade
10. Data wiring — confirm paper_trades.json updates correctly
